What is a Servlet

A servlet is a Java technology-based Web component, managed by a container, that generates dynamic content.

Servlets interact with Web clients via a request/response paradigm implemented by the servlet container.

Servlet Container

Servlet containers execute and manage servlets.

The servlet container is a part of a Web server or application server that provides the network services over which requests and responses are sent, decodes MIME-based requests, and formats MIME-based responses. A servlet container also contains and manages servlets through their lifecycle.

A servlet container can be built into a host Web server, or installed as an add-on component to a Web Server via that serverâ€™s native extension API. Servlet containers can also be built into or possibly installed into Web-enabled application server

The request object encapsulates all information from the client request.

You can get these information from request object

  • Request URL
  • Parameters

ServletResponse

The response object encapsulates all information to be returned from the server to the client.

Servlet Lifecycle

  • Loading and Instantiation - locate the class and construct a instance of that class
  • Initialization - init with ServletConfig configured in DD, before it can handle any request
  • Request Handling - handle each request (in a seperate Thread) and modify response
  • End of Service - waiting for all the request handled and cleanup resources (destory)
